# Ограниченные контексты (Bounded Contexts)

Границы между частями домена для независимой разработки и чёткой ответственности.

## Контексты

| Context                  | Тип домена          | Ответственность                                          |
| ------------------------ | ------------------- | -------------------------------------------------------- |
| **Customer**             | Supporting          | Идентификация клиента, договорные отношения              |
| **Wallet**               | Core                | Управление кошельками, балансы, резервации               |
| **Payments**             | Core                | Жизненный цикл платежа от создания до финального статуса |
| **Core Banking**         | Supporting (legacy) | Главная книга, бухгалтерские проводки                    |
| **Anti-Fraud / Scoring** | Core                | Fraud-detection, оценка рисков                           |
| **Notifications**        | Generic             | Уведомления клиентам (SMS/Push/Email)                    |
| **Callbacks**            | Supporting          | Callback от провайдеров (in) и мерчантам (out)           |
| **Query**                | Supporting          | CQRS read model для проекций платежей                    |
